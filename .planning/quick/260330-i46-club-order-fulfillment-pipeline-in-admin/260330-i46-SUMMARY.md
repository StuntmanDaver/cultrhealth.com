## What Changed

### 1. Database Migration (`migrations/033_club_order_fulfillment.sql`)
Added fulfillment tracking columns to `club_orders`:
- `paid_at`, `shipped_at` (existed from 032), `fulfilled_at` — timestamps for each stage
- `tracking_carrier`, `tracking_number`, `tracking_url` — shipping details

### 2. Status Update API (`app/api/admin/club-orders/[orderId]/status/route.ts`)
Rewrote existing route with proper status transitions:
- `approved`/`invoice_sent` → `paid` (sets `paid_at`)
- `paid` → `shipped` (sets `shipped_at` + tracking fields) or `fulfilled`
- `shipped` → `fulfilled` (sets `fulfilled_at`)
- Atomic updates with concurrent-safety (`AND status = currentStatus`)

### 3. Club Orders List API (`app/api/admin/club-orders/route.ts`)
- Now returns tracking columns (`paid_at`, `shipped_at`, `fulfilled_at`, `tracking_*`)
- Improved sort order: pending → invoice_sent → paid → shipped → fulfilled
- Increased limit to 200

### 4. Analytics API (`app/api/admin/analytics/route.ts`)
- Added `getClubOrderFulfillmentCounts()` to analytics response
- New `clubOrderFulfillment` field in `AnalyticsData` type

### 5. DB Functions (`lib/db.ts`)
- `updateClubOrderStatus()` — generic status updater with transition validation
- `getClubOrderFulfillmentCounts()` — counts per status for pipeline visualization

### 6. Fulfillment Pipeline UI (`app/admin/orders/PendingApprovalTab.tsx`)
Complete rewrite from approval-only to full lifecycle management:
- **Pipeline visualization bar** — clickable stage counts with arrows
- **Status filter pills** — All / Active / Cancelled / Completed + per-stage
- **Context-aware action buttons** per order:
  - Pending → "Approve & Send Invoice"
  - Invoice Sent → "Mark Paid"
  - Paid → "Mark Shipped" (with tracking form) or "Mark Fulfilled (no shipping)"
  - Shipped → "Mark Fulfilled"
- **Order timeline** — green dots with timestamps for each lifecycle stage
- **Tracking display** — carrier + tracking number badge with "Track Package" link
