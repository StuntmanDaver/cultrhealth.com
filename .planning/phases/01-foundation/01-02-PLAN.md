  <action>
    Create `migrations/020_siphox_tables.sql` with all 3 tables in a single migration file. Follow the exact schema from RESEARCH.md with these specifications:

    **Table 1: siphox_customers** (customer mapping)
    - `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
    - `phone_e164` VARCHAR(20) NOT NULL -- links to portal_sessions.phone_e164
    - `siphox_customer_id` VARCHAR(100) NOT NULL -- SiPhox API _id
    - `external_id` VARCHAR(50) -- same as phone_e164, stored for reference
    - `first_name` VARCHAR(100)
    - `last_name` VARCHAR(100)
    - `email` VARCHAR(255)
    - `created_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - `updated_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - UNIQUE index on `phone_e164`
    - UNIQUE index on `siphox_customer_id`

    **Table 2: siphox_kit_orders** (kit order tracking)
    - `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
    - `siphox_customer_id` VARCHAR(100) NOT NULL REFERENCES siphox_customers(siphox_customer_id)
    - `siphox_order_id` VARCHAR(100) NOT NULL -- SiPhox API order _id
    - `kit_type` VARCHAR(100) NOT NULL
    - `quantity` INTEGER NOT NULL DEFAULT 1
    - `status` VARCHAR(50) NOT NULL DEFAULT 'ordered' -- ordered, shipped, registered, processing, results_ready
    - `tracking_number` VARCHAR(200)
    - `stripe_subscription_id` VARCHAR(255) -- links to Stripe checkout that triggered this order
    - `is_test_order` BOOLEAN NOT NULL DEFAULT false
    - `created_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - `updated_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - UNIQUE index on `siphox_order_id`
    - Index on `siphox_customer_id`

    **Table 3: siphox_reports** (cached biomarker reports, immutable)
    - `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
    - `siphox_customer_id` VARCHAR(100) NOT NULL REFERENCES siphox_customers(siphox_customer_id)
    - `siphox_report_id` VARCHAR(100) NOT NULL -- SiPhox API report _id
    - `report_data` JSONB NOT NULL -- Zod-validated biomarker results
    - `suggestions` JSONB -- SiPhox suggestions array
    - `report_status` VARCHAR(50) -- e.g. "completed"
    - `fetched_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - `created_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - UNIQUE index on `siphox_report_id`
    - Index on `siphox_customer_id`

    Add a comment header explaining this migration's purpose. Use `CREATE TABLE IF NOT EXISTS` and `CREATE INDEX IF NOT EXISTS` for idempotency (following existing migration patterns).
