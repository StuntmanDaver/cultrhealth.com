# Plan 05-01: cultrclub.com Production Go-Live

## Objective

Point `cultrclub.com` to Cloudflare Pages production branch and verify the live site is working correctly. Gate: all 11 staging validation checks must have passed in Phase 4.

## Context

- **Source plan:** `/Users/davidk/.claude/plans/snazzy-humming-treasure.md` (Phase 8 Steps 4-5)
- **Prerequisite:** All 11 checks in 04-02 must be PASS
- **Note:** `autonomous: false` — requires Cloudflare dashboard + DNS changes

## Tasks

### Task 1: Register/transfer cultrclub.com to Cloudflare DNS

If `cultrclub.com` is not already in Cloudflare:
1. Register the domain via Cloudflare Registrar, OR
2. Transfer DNS management to Cloudflare (update nameservers at registrar)

### Task 2: Add cultrclub.com to Cloudflare Pages project

In **Cloudflare Pages** → cultrclub project → Custom Domains:
1. Add domain: `cultrclub.com`
2. Assign to: `main` branch (production)
3. Cloudflare Pages auto-configures DNS and SSL

Also add `www.cultrclub.com` → redirect to `cultrclub.com` (Cloudflare handles www redirect automatically).

### Task 3: Verify SSL

Wait for Cloudflare to issue SSL certificate (usually < 2 minutes with Cloudflare DNS). Verify:
```
curl -I https://cultrclub.com
```
Should return HTTP 200 with valid SSL.

### Task 4: Confirm production env vars

Verify in Cloudflare Pages → Settings → Environment Variables (Production):
- `NEXT_PUBLIC_SITE_URL` = `https://cultrclub.com`
- `ADMIN_BASE_URL` = `https://cultrhealth.com`
- All other 15 vars set correctly

### Task 5: Run smoke test on production

Quick verification on `https://cultrclub.com`:
- [ ] Home page loads correctly
- [ ] `https://cultrclub.com/api/health` returns `{ ok: true }`
- [ ] Signup form is visible and Turnstile widget loads
- [ ] No console errors related to missing env vars

### Task 6: Monitor for 24 hours

Watch Cloudflare Pages analytics and error logs for:
- 5xx errors
- Build failures
- Any @vercel/postgres import errors (would indicate missed swap)
