### Task 5: Set up join.cultrhealth.com → cultrclub.com redirect in Cloudflare DNS

In **Cloudflare DNS** (cultrhealth.com zone), add a redirect rule:

Option A: Cloudflare Page Rule (legacy):
- URL: `join.cultrhealth.com/*`
- Setting: Forwarding URL (301 Permanent Redirect) → `https://cultrclub.com/$1`

Option B: Cloudflare Redirect Rule (modern):
- When: hostname equals `join.cultrhealth.com`
- Then: Dynamic redirect → `https://cultrclub.com${uri.path}` (301)

This preserves any bookmarks or email links to `join.cultrhealth.com`.

### Task 6: Verify redirect

```bash
curl -I https://join.cultrhealth.com
```
Expected: `HTTP/1.1 301` with `Location: https://cultrclub.com`

### Task 7: Schedule Vercel domain alias removal

In **Vercel Dashboard** → cultrhealth.com production deployment → Domains:
- Remove `join.cultrhealth.com` custom domain alias

**Wait 2-4 weeks** after the Cloudflare redirect is confirmed working before removing this alias. This prevents broken links during the DNS propagation window.

Note this as a calendar reminder — do not remove Vercel alias on the same day as the redirect.
