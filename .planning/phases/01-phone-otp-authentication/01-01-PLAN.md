    - verifyPortalAccessToken(token) returns PortalSession when valid, null when expired or invalid
    - verifyPortalRefreshToken(token) returns PortalSession when valid, null when expired or invalid
    - verifyPortalAccessToken rejects tokens where type != 'portal_access' (e.g., rejects a refresh token)
    - verifyPortalRefreshToken rejects tokens where type != 'portal_refresh' (e.g., rejects an access token)
    - Portal tokens use SESSION_SECRET (same secret as existing auth, via process.env.SESSION_SECRET)
    - Portal cookie names are 'cultr_portal_access' and 'cultr_portal_refresh' (NOT 'cultr_session')

    - Refresh cookie path is '/api/portal/refresh' (scoped to refresh endpoint only)
    - clearPortalCookies deletes both portal cookies
    - getPortalSession reads access token from cookies and returns PortalSession or null
    - verifyPortalAuth(request: NextRequest) reads portal access token from request cookies, returns { authenticated, phone, asherPatientId }
    - upsertPortalSession(phone, phoneE164, patientId, firstName?, lastName?) inserts or updates portal_sessions row
    - getPortalSessionByPhone(phoneE164) returns matching row or null

    **Step 1: Create migration file `migrations/014_portal_sessions.sql`**

    Create the portal_sessions table with columns:
    - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
    - phone VARCHAR(20) NOT NULL (display format)
    - phone_e164 VARCHAR(20) NOT NULL (E.164 format, used for lookups)
    - asher_patient_id INTEGER (nullable -- may be null if patient not yet in Asher Med)
    - first_name VARCHAR(100)
    - last_name VARCHAR(100)
    - verified_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - last_login_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()

    Create unique index on phone_e164 and regular index on asher_patient_id.

    **Step 2: Create `lib/portal-auth.ts`**

    Follow the pattern in `lib/auth.ts` but keep completely separate (AUTH-06). Key decisions:
    - Use `jose` library (already installed) for JWT sign/verify
    - Use `process.env.SESSION_SECRET` for the signing key (same secret, but different token types prevent cross-use)
    - Cookie names: `cultr_portal_access` and `cultr_portal_refresh`
    - Access token maxAge: 900 seconds (15 minutes)
    - Refresh token maxAge: 604800 seconds (7 days)
    - Refresh cookie `path: '/api/portal/refresh'` (security: only sent to refresh endpoint)
    - Access cookie `path: '/'`
    - Both cookies: httpOnly=true, secure=(NODE_ENV==='production'), sameSite='lax'

    Export the PortalSession interface:
    ```typescript
    export interface PortalSession {
      phone: string
      asherPatientId: number | null
    }
    ```

    Export constants for cookie names: `PORTAL_ACCESS_COOKIE = 'cultr_portal_access'`, `PORTAL_REFRESH_COOKIE = 'cultr_portal_refresh'`

    Functions to implement:
    - `createPortalAccessToken(phone: string, patientId: number | null): Promise<string>`
    - `createPortalRefreshToken(phone: string, patientId: number | null): Promise<string>`
    - `verifyPortalAccessToken(token: string): Promise<PortalSession | null>`
    - `verifyPortalRefreshToken(token: string): Promise<PortalSession | null>`
    - `setPortalCookies(accessToken: string, refreshToken: string): Promise<void>`
    - `clearPortalCookies(): Promise<void>`
    - `getPortalSession(): Promise<PortalSession | null>` (reads access cookie, returns session)
    - `verifyPortalAuth(request: NextRequest): Promise<{ authenticated: boolean; phone: string | null; asherPatientId: number | null }>`

    The `verifyPortalAuth` function reads the portal access cookie from `request.cookies` (not from `next/headers` cookies) for use in API route handlers.
