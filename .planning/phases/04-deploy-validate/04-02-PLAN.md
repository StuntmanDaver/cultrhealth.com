# Plan 04-02: Staging Validation

## Objective

Run all 11 validation checks against `staging.join.cultrhealth.com` before authorizing production cutover to `cultrclub.com`. Every check must pass.

## Context

- **Source plan:** `/Users/davidk/.claude/plans/snazzy-humming-treasure.md` (Phase 8 Step 3, Verification section)
- **Staging URL:** `https://staging.join.cultrhealth.com`
- **Note:** `autonomous: false` — requires human testing

## Validation Checklist

Each check must be manually verified and marked as PASS before proceeding to Phase 5.

### Check 1: Club signup

**Steps:**
1. Go to `https://staging.join.cultrhealth.com`
2. Complete the club signup form with a test email
3. Submit

**Pass criteria:**
- [ ] `club_members` row created in Neon DB (check via Neon console or admin dashboard)
- [ ] Welcome email arrives with `cultrclub.com` link (not `join.cultrhealth.com`)

### Check 2: Order submission

**Steps:**
1. Sign up or use an existing test account
2. Add a product to cart and submit an order

**Pass criteria:**
- [ ] `club_orders` row created in Neon DB with correct amounts and tax
- [ ] Tax amount is correct (7.5% FL rate if applicable)

### Check 3: Admin approval email

**Steps:**
1. Submit a test order (from Check 2)
2. Check the admin approval email

**Pass criteria:**
- [ ] Approval link URL starts with `staging.cultrhealth.com/api/admin/club-orders/...` (NOT cultrclub.com)
- [ ] Admin panel link goes to `staging.cultrhealth.com/admin`

### Check 4: Admin dashboard shows the order

**Steps:**
1. Go to `https://staging.cultrhealth.com/admin`
2. Navigate to Club Orders tab

**Pass criteria:**
- [ ] The test order from Check 2 appears in the Club Orders list
- [ ] Order status, amounts, and member info are correct

### Check 5: Inventory decrement

**Steps:**
1. Check `product_inventory.stock_quantity` for the ordered product before and after order submission

**Pass criteria:**
- [ ] `product_inventory.stock_quantity` decrements by the ordered quantity after order

### Check 6: UTM cookie

**Steps:**
1. Visit `https://staging.join.cultrhealth.com?utm_source=test&utm_medium=email`
2. Inspect browser cookies

**Pass criteria:**
- [ ] `cultr_visitor_ctx` cookie exists
- [ ] Cookie domain is `.cultrclub.com` (not `.cultrhealth.com`)
- [ ] Cookie contains `s: "test"`, `m: "email"`

### Check 7: Returning member recognition

**Steps:**
1. Sign up and receive the `cultr_club_visitor` cookie
2. Return to `https://staging.join.cultrhealth.com` in a new tab (with cookie present)

**Pass criteria:**
- [ ] Page shows the recognized member state (member name, personalized greeting, or similar)

### Check 8: Coupon codes

**Steps:**
1. At checkout, apply the OWNER coupon code
2. Apply the CULTR10 hardcoded coupon
3. Apply an active affiliate code

**Pass criteria:**
- [ ] All three coupon types apply correctly and show correct discount
- [ ] Affiliate attribution recorded if applicable

### Check 9: Bundle discount

**Steps:**
1. Add two bundled products to cart (GHK-CU + Glutathione)

**Pass criteria:**
- [ ] 10% bundle discount applied automatically

### Check 10: Creator attribution

**Steps:**
1. Visit with a `cultr_attribution` cookie set (or click a tracking link)
2. Complete an order

**Pass criteria:**
- [ ] Commission record created in `commission_ledger` after order

### Check 11: Rate limiting

**Steps:**
1. Submit the signup form 6+ times in the same hour from the same IP

**Pass criteria:**
- [ ] 6th submission returns HTTP 429 with rate limit message

### Check 12: Health endpoint

**Steps:**
1. `curl https://staging.join.cultrhealth.com/api/health`

**Pass criteria:**
- [ ] Returns `{ "ok": true }` with HTTP 200

## Go/No-Go Decision

After all checks:
- **All 11 pass** → Proceed to Phase 5 (Production Cutover)
- **Any check fails** → Fix the issue, re-deploy to staging, re-run the failing check before proceeding
