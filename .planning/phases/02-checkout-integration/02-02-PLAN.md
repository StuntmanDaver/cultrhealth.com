    **Checkout Session route (app/api/checkout/subscription/route.ts):**
    Create a new POST route:
    - Accept `{ planSlug: string, email: string }` body
    - Validate planSlug is 'core' (this route only handles Core; other tiers keep Payment Links)
    - Validate email is present
    - Initialize Stripe with same pattern as webhook route (`new Stripe(process.env.STRIPE_SECRET_KEY || '', { apiVersion: '2026-02-25.clover' })`)
    - Look up Core plan from PLANS array to get stripePriceId
    - Read attribution cookie from request cookies (`cultr_attribution`)
    - Build Checkout Session:
      - `mode: 'subscription'`
      - `customer_email: email`
      - `line_items: [{ price: PLANS.find(p => p.slug === 'core').stripePriceId, quantity: 1 }]`
      - Try `optional_items` first (blood test as optional one-time add-on). If Stripe throws an error (API version doesn't support it), fall back to adding blood test as a regular `line_item` with `adjustable_quantity: { enabled: true, minimum: 0, maximum: 1 }`.
      - `metadata: { plan_tier: 'core' }`
      - `client_reference_id: attributionCookie ? 'attr_' + attributionCookie : undefined`
      - `success_url: '{siteUrl}/success?session_id={CHECKOUT_SESSION_ID}'`
      - `cancel_url: '{siteUrl}/pricing'`
      - `allow_promotion_codes: true` (so existing FOUNDER15, FIRSTMONTH coupons work)
    - Return `{ success: true, redirectUrl: session.url }` (Checkout Sessions return a hosted URL)
    - Rate limit using same pattern as existing checkout route (apiLimiter)

    **Checkout route modification (app/api/checkout/route.ts):**

All other tiers (catalyst, concierge, club) continue returning their Payment Link URL as before.
