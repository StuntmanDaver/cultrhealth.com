<objective>
Wire SiPhox kit ordering into the Stripe checkout pipeline. When a Catalyst+ or Concierge member subscribes, a kit order is created automatically. If the address or API is unavailable, the order is deferred and a cron job retries every 15 minutes. Refunds trigger support notifications. The subscription always activates regardless of SiPhox outcome.

Purpose: Automates the blood test kit ordering that was previously manual, ensuring every eligible member gets a kit without staff intervention.
Output: Fulfillment orchestration module, extended database schema, cron retry route, webhook extensions, 3 new email templates, and comprehensive tests.
</objective>

    Create the orchestration module with 4 exported functions + 2 internal helpers:

    `resolveShippingAddress(customerEmail: string): Promise<ShippingAddress | null>` (internal)
    - Query asher_orders + pending_intakes by email (case-insensitive) for the latest intake_data
    - Extract shippingAddress from intake_data JSONB: { address1 -> street1, address2 -> street2, city, stateAbbreviation -> state, zipCode -> zip, country defaults to 'US' }
    - Also extract phone and name from intake_data for SiPhox customer creation
    - Return null if no intake data found

    `resolveMemberData(customerEmail: string): Promise<MemberData | null>` (internal)
    - Wraps resolveShippingAddress plus extracts firstName, lastName, phone from the same intake_data
    - Returns { firstName, lastName, phone, email, address } or null

    `triggerSiphoxFulfillment(params: { customerEmail, planTier, stripeCheckoutSessionId, stripeSubscriptionId? }): Promise<void>`
    - **Tier check first:** Only proceed if planTier is 'catalyst', 'concierge', or ('core' with has_blood_test_addon metadata -- detected by checking Stripe session line items in the webhook caller before invoking this function). If tier is 'club' or 'core' without add-on, return immediately.
    - **Idempotency:** Check getOrderByCheckoutSession(stripeCheckoutSessionId). If row exists, return early.
    - **Resolve address:** Call resolveMemberData(customerEmail). If null, insert kit order with status 'pending_intake' via insertFulfillmentOrder, return.
    - **Check credits:** Call checkCreditBalance(). If balance < 1, insert with status 'needs_credits', send sendLowCreditAlert (already exists), return.
    - **Create SiPhox customer:** Use createCustomer or getCustomerByExternalId (use phone as external_id per Phase 1 pattern). Use upsertSiphoxCustomer to persist mapping.
    - **Create SiPhox order:** Call createOrder with the recipient address. Use kit_type 'standard_panel' (the SiPhox kit type).
    - **On success:** Insert kit order with status 'fulfilled' and the real siphox_order_id.
    - **On API error:** Catch SiphoxApiError. If status is 402 or credit-related, insert with 'needs_credits'. Otherwise insert with 'pending_fulfillment' for cron retry.
    - **Send confirmation email:** On 'fulfilled' status, call sendKitFulfillmentEmail with member name, email, and address.
    - Entire function wrapped in try/catch that never throws -- logs errors only.

    `processDeferredOrders(): Promise<{ processed: number; fulfilled: number; stillPending: number }>`
    - Get all pending_intake orders via getDeferredIntakeOrders()
    - For each: try resolveShippingAddress. If address now available, attempt full SiPhox fulfillment (createCustomer + createOrder). Update status to 'fulfilled' or 'pending_fulfillment' (if API fails). If address still missing, skip.
    - Return counts.

    `retryFailedOrders(): Promise<{ retried: number; fulfilled: number; permanentlyFailed: number }>`
    - Get all pending_fulfillment orders via getPendingFulfillmentOrders() (already filters retry_count < 3)
    - For each: attempt SiPhox createOrder (customer should already exist from initial attempt). On success, updateFulfillmentStatus to 'fulfilled'. On failure, incrementRetryCount. If retry_count reaches 3, updateFulfillmentStatus to 'failed' and call sendSiphoxFailureAlert.
    - Return counts.

    `notifySiphoxRefund(params: { stripeCheckoutSessionId, refundAmount, customerEmail?, customerName? }): Promise<void>`
    - Look up kit order by getOrderByCheckoutSession(stripeCheckoutSessionId). If not found, return (not every checkout has a SiPhox order).
    - Call sendSiphoxRefundAlert with: member name/email, plan_tier, siphox_order_id, fulfillment_status, refund amount, suggested action (based on status -- if 'pending_intake' suggest "Cancel order, kit not yet shipped"; if 'fulfilled' suggest "Kit already shipped, consider requesting return").
    - Never throw. Per user decision: never auto-cancel SiPhox orders on refund.

    // SiPhox kit fulfillment (non-fatal -- subscription activates regardless)
    try {
      const { triggerSiphoxFulfillment } = await import('@/lib/siphox/fulfillment')
      const planTier = session.metadata?.plan_tier || 'unknown'

      // For Core tier, check if blood test add-on was selected
      // by examining the session line items for the blood test price
      let hasBloodTestAddon = false
      if (planTier === 'core') {
        const stripe = getStripe()
        const lineItems = await stripe.checkout.sessions.listLineItems(session.id)
        hasBloodTestAddon = lineItems.data.some(item =>
          item.price?.id === process.env.BLOOD_TEST_STRIPE_PRICE_ID
        )
      }

      // Only trigger for eligible tiers
      const isEligible = planTier === 'catalyst' || planTier === 'concierge' || (planTier === 'core' && hasBloodTestAddon)
      if (isEligible) {
        await triggerSiphoxFulfillment({
          customerEmail: session.customer_details?.email || '',
          planTier,
          stripeCheckoutSessionId: session.id,
          stripeSubscriptionId: typeof session.subscription === 'string' ? session.subscription : session.subscription?.id,
        })
      }
    } catch (siphoxError) {
      console.error('SiPhox fulfillment failed (non-fatal):', siphoxError)

    // SiPhox refund notification (non-fatal)
    try {
      const { notifySiphoxRefund } = await import('@/lib/siphox/fulfillment')
      // Find the checkout session that originated this charge
      // The charge -> payment_intent -> checkout_session linkage:
      // We need to look up the checkout session by the payment intent
      const stripe = getStripe()
      const paymentIntentId = typeof charge.payment_intent === 'string' ? charge.payment_intent : charge.payment_intent?.id
      if (paymentIntentId) {
        const sessions = await stripe.checkout.sessions.list({ payment_intent: paymentIntentId, limit: 1 })
        if (sessions.data[0]) {
          await notifySiphoxRefund({
            stripeCheckoutSessionId: sessions.data[0].id,
            refundAmount: (charge.amount_refunded || 0) / 100,
            customerEmail: charge.billing_details?.email || undefined,
            customerName: charge.billing_details?.name || undefined,
          })
