  <behavior>
    - GET returns 401 when not authenticated
    - GET returns 401 when asherPatientId is null (cannot list without patient)
    - GET queries asher_uploaded_files by asher_patient_id, ordered by uploaded_at DESC
    - GET generates preview URL for each document via getPreviewUrl (best-effort, null if preview fails)
    - GET returns documents array with id, purpose, contentType, previewUrl, uploadedAt
    - GET returns empty array when no documents exist (not an error)
    - POST returns 401 when not authenticated
    - POST returns 401 when asherPatientId is null
    - POST validates contentType against allowed list -- returns 400 for invalid types
    - POST validates purpose against allowed list -- returns 400 for invalid purpose
    - POST calls getPresignedUploadUrl(contentType) from Asher Med
    - POST inserts record into asher_uploaded_files with s3_key, content_type, file_purpose, asher_patient_id
    - POST returns { success: true, uploadUrl, key }
    - POST returns 502 when Asher Med getPresignedUploadUrl throws
    - POST supports mock mode on staging (same pattern as intake upload route)
  </behavior>

    GET handler:

    - Query: `SELECT id, s3_key, content_type, file_purpose, uploaded_at FROM asher_uploaded_files WHERE asher_patient_id = ${auth.asherPatientId} ORDER BY uploaded_at DESC`

    - Mock mode check: if no `ASHER_MED_API_KEY` and staging/dev, generate mock key `mock/portal/${purpose}/${Date.now()}-${random}`, insert into DB, return mock response
    - Production: call `getPresignedUploadUrl(contentType)`, insert into `asher_uploaded_files` (s3_key, content_type, file_purpose, asher_patient_id), return `{ success: true, uploadUrl, key }`
    - Catch errors -> 502
