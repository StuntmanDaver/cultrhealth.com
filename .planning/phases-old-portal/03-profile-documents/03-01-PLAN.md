  <name>Task 1: Profile API route (GET patient data, PUT address update) with tests</name>
  <files>tests/api/portal-profile.test.ts, app/api/portal/profile/route.ts</files>
  <behavior>
    - GET returns 401 when not authenticated (mockVerifyPortalAuth returns { authenticated: false })
    - GET returns { success: true, profile: null } when asherPatientId is null (Case C user)
    - GET returns personal info (firstName, lastName, email, phone, dateOfBirth, gender) from Asher Med patient
    - GET returns address object (address1, address2, city, state mapped from stateAbbreviation, zipCode mapped from zipcode) from Asher Med patient
    - GET returns measurements (height in inches, weight in lbs, bmi) from Asher Med patient, null when missing
    - GET returns 502 when Asher Med API throws
    - PUT returns 401 when not authenticated
    - PUT returns 401 when asherPatientId is null (cannot update without patient)
    - PUT validates required address fields (address1, city, state, zipCode) -- returns 400 if missing
    - PUT validates state is a valid US state abbreviation -- returns 400 if invalid
    - PUT calls updatePatient with correct Asher Med field names (stateAbbreviation, zipcode -- NOT state, zipCode)
    - PUT returns { success: true } on successful update
    - PUT returns 502 when Asher Med updatePatient throws
  </behavior>
  <action>
    Write tests FIRST following the portal-orders.test.ts mock pattern:
    - `// @vitest-environment node` header
    - Mock `next/server` with MockNextResponse, mock `@/lib/portal-auth`, mock `@/lib/asher-med-api` (getPatientById + updatePatient)
    - Create test request helper: `makeRequest(method, body?)` returning plain object with cookies mock
    - Group tests: `describe('GET /api/portal/profile')` and `describe('PUT /api/portal/profile')`

    Then implement `app/api/portal/profile/route.ts`:
    - `export const dynamic = 'force-dynamic'` at top
    - Import `verifyPortalAuth` from `@/lib/portal-auth`
    - Import `getPatientById`, `updatePatient` from `@/lib/asher-med-api`
    - Import `US_STATES` from `@/lib/config/asher-med` for state validation in PUT
    - Import `z` from `zod` for PUT body validation

    GET handler:
    - Verify auth, return 401 if not authenticated
    - Return `{ success: true, profile: null }` if no asherPatientId
    - Call `getPatientById(auth.asherPatientId)`
    - Shape response: personal info fields flat, address as nested object mapping `stateAbbreviation` -> `state` and `zipcode` -> `zipCode`, measurements as nested object with null fallbacks
    - Catch errors -> 502

    PUT handler:
    - Verify auth, return 401 if not authenticated or no asherPatientId
    - Parse body, extract `address` object
    - Validate with Zod schema: address1 (string, min 1), city (string, min 1), state (string, 2 chars, must be in US_STATES), zipCode (string, 5 chars regex /^\d{5}$/), address2 optional
    - Call `updatePatient(auth.asherPatientId, { address1, address2: address2 || null, city, stateAbbreviation: state, zipcode: zipCode, country: 'US' })`
    - Return `{ success: true }`
    - Catch Zod errors -> 400, Asher Med errors -> 502
  </action>
