<objective>
Build the dashboard client UI that replaces the placeholder page with a status-first layout: hero card for active order, compact order list, Stripe-style slide-over detail panel, empty state for new members, and quick links section.

Purpose: This is the first real data screen in the portal. Members should see their order status immediately on login without calling support or checking email. The hero card pattern puts the most important information front and center.

Output: `app/portal/dashboard/page.tsx` (server metadata wrapper), `app/portal/dashboard/DashboardClient.tsx` (full dashboard UI), `tests/components/PortalDashboard.test.tsx` (component tests).

       - Status badge: `<span>` with `bgClass` and `textClass` from `getStatusDisplay()`, plus the `label` text. Rendered as a pill (`rounded-full px-3 py-1 text-sm font-medium`)

       - Date info: "Ordered {formatted createdAt}" and "Updated {formatted updatedAt}" using `new Date(date).toLocaleDateString('en-US', { month: 'short', day: 'numeric', year: 'numeric' })`
       - Provider indicator: If `doctorId` is not null, show "Provider assigned" with a check icon; if null, show "Awaiting provider" with a clock icon
       - Card styling: `rounded-2xl border border-brand-primary/10 bg-white p-6 shadow-sm cursor-pointer hover:shadow-md transition-shadow`
       - Clicking the hero card opens the slide-over for that order

    3. **Order list** -- Orders excluding the hero card order, rendered as compact rows:
       - Each row: medication name, status badge (smaller pill), formatted date
       - Row styling: `rounded-xl bg-white p-4 border border-brand-primary/5 cursor-pointer hover:bg-brand-primary/[0.02] transition-colors`
       - Clicking a row sets `selectedOrderId` state which triggers the slide-over

    4. **Slide-over panel** -- Right-side panel (Stripe-style):

       - When `selectedOrderId` is set: `translate-x-0`; otherwise: `translate-x-full`
       - On open: fetch `/api/portal/orders/${selectedOrderId}` for fresh data. Show loading spinner briefly. Display all fields: medication name, status badge + explanation, order date, last updated, provider assignment ("Provider assigned" or "Awaiting provider"), order ID (formatted as `#${id}`), order type, partner notes (if any)
       - Close button: X icon in top-right corner
       - When detail panel closes, update the specific order in the local orders list state with the freshly fetched data (prevents stale status pitfall from research)
