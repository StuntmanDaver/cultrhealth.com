<objective>
Build the data layer for the portal dashboard: a shared status mapping utility and two API proxy routes that fetch order data from Asher Med through the portal auth system.

Purpose: The dashboard UI (Plan 02) needs typed order data and consistent status display. These API routes keep the Asher Med API key server-side and verify portal authentication before every request. The status mapping utility centralizes the label/explanation/color logic used across hero card, list rows, and detail panel.

Output: `lib/portal-orders.ts` (types + status mapping), `/api/portal/orders` (list), `/api/portal/orders/[id]` (detail), plus comprehensive tests for all three.
</objective>
